前言

本博客已迁往vultr，另外关于https证书的获取和配置也已更新，请参看：《博客迁移说明》

本博客搭建在搬瓦工的vps上，之前用的都是http，一直想升级到https，但都没付诸实践，这次正好尝试一下，并做一个总结。首先，需要了解https协议的一些原理，阮一峰的《HTTPS 协议概述》 《图解SSL/TLS协议》这几篇文章中有详细的介绍。其次，本文有部分引用了 Jerry Qu 的《Let’s Encrypt，免费好用的 HTTPS 证书》 一文。再次，由于我还是一个小白，文中难免有错误之处，欢迎指正。

关于证书

由于是个人网站，我准备使用 Let’s Encrypt 这一免费证书，这也是当前最为流行的免费证书，而且配置过程简单，虽然每次只有90天的有效期，但可以部署脚本定期更新，所以还是比较方便的。Let’s Encrypt 使用ACME协议来签发证书，并提供了官方的客户端工具 Certbot，但是我使用了 acme-tiny 这个更为小巧的开源工具。

开始申请

创建目录

我使用的 Web Server 是 nginx，进入 nginx 的配置目录conf，创建目录ssl用来存放证书申请过程中的各种临时和证书文件，进入这个目录，之后的操作都在这个目录中进行。

创建账号私钥

我们用公钥在 Let’s Encrypt 注册账号，然后用对应的私钥对请求签名。因此，先要生成一个 key，acme-tiny 会使用这个key来注册账号并对请求签名。

为了证明你对需要申请证书的域名有控制权，Let’s Encrypt 需要你在该域名下配置验证文件，再通过创建 CSR 时指定的域名访问，如果可以访问则表明你对这个域名有控制权。acme-tiny 这个脚本会在你指定的目录下生成这些验证文件，因此你需要做的就是保证能够通过 “.well-known/acme-challenge/” 这个 url 路径能够访问到这些文件。
