前言

之前博客一直挂在搬瓦工上，不得不说搬瓦工十分稳定，但是访问速度却让人非常捉急，在经过诸多尝试后，决定迁往vultr。vultr的管理界面十分简洁，看起来很舒服，而且也支持支付宝和微信支付，更重要的一点是vultr是按时付费的，觉得用起来不爽可以直接随时停掉。

之前博客也申请并配置了https证书，见前文《网站升级https记录》。但是在尝试使用过 Let’s Encrypt 官方的客户端 Certbot 后，考虑到易用性和稳定性，决定更新一篇记录，方便自己查看。

关于https协议

https协议的一些原理，在阮一峰的《HTTPS 协议概述》 《图解SSL/TLS协议》这几篇文章中有详细的介绍。这里不再赘述，其实也讲不明白。

介绍Certbot

Certbot 是一个十分易用的，可以从 Let’s Encrypt 获取证书并部署证书到服务器上的一个工具。申请和管理证书通常是十分麻烦和复杂的，但是 Certbot 简化了这些步骤，提供了一些简单的命令来帮助我们完成这些操作。

使用 Certbot 工具需要具备简单的linux知识，包括但不限于：

  • 通过ssh登录服务器
  • 通过终端输入命令
  • 了解web服务器的简单配置
  • 在域名提供商提供的管理后台修改域名的解析

Certbot安装

基本上主流的 UNIX-like 操作系统都提供了对应的 Certbot 包，可以通过包管理器下载和安装 Certbot。

安装 Certbot 需要注意以下几点要求（关于插件和Certbot-auto下文会提到）：

  • 系统已安装和运行 Python 2.7 或 3.4+
  • 需要 root 权限写入/etc/letsencrypt, /var/log/letsencrypt, /var/lib/letsencrypt这几个目录;绑定80和443端口（如果你使用了standalone插件）以及读取和修改web服务器配置
  • 如果使用Apache插件，需要系统安装有1.0版本的augeas。目前此插件支持基于 Debian, Fedora, SUSE, Gentoo and Darwin 的主流操作系统。
  • 安装Certbot-auto需要至少512M的内存以构建一些依赖，而通过包管理器安装的 Certbot 则没有此要求。

如果你是离线安装或者你使用的操作系统没有提供相应的 Certbot 安装包，那么你就可以使用替代方案来安装 Certbot。比如：从源码安装或是使用Certbot-auto。这里不做详细介绍了，感兴趣的直接去看官网吧。

Certbot使用

Certbot 使用两种插件（authenticators 和 installers）分别来获取和安装证书。

authenticators 插件通过 certbot certonly 命令来获取证书。包括以下过程：验证持有网站、获取指定域名的证书、将证书放置到 /etc/letsencrypt 目录。该插件不会安装证书（它不会修改任何web服务器的配置文件）。如果你在执行获取证书命令时指定了多个域名，那这些域名将被放置到一个证书中。如果你想为指定域名获取单独的证书，那你就需要分开多次执行 Certbot 获取证书命令。

Installers 插件通过 certbot install 命令来安装证书，也就是会修改web服务器的配置文件。

Certbot 也提供了可以同时获取证书并安装证书的插件（你可以理解为这种插件是前面两种插件的集合体），使用这样的插件需要执行 certbot run命令并指定插件，例如certbot run --nginx。

此外，插件使用ACME协议中的一种来验证你持有网站。这些选择分别为： http-01（通过80端口）， tls-sni-01 (443端口) 和 dns-01 (通过验证DNS记录)。一些插件支持多种验证类型，你可以通过 --preferred-challenges选项来决定使用哪种。

因为 Let’s Encrypt 颁发的证书只有90天的有效期，所以必须定期更新，以免证书失效。更新证书的方法也很简单，使用certbot renew命令即可，但是对于前面采取manual方式获取的通配符证书，因为要验证dns，则不能简单的使用此命令来更新证书，还是必须使用前面的步骤来更新证书。
